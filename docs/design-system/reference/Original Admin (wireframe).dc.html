Little Scout · Admin console · web-only operator intranet
From a reporting dashboard to an operator's cockpit
Built on your real admin theme — forest green #2d5a3d, amber accent, Outfit. Today's console reports data well but lets an operator do almost nothing. With seeding wound down, the daily job becomes moderation, data quality, and user trust — so the redesign leads with those.
What the console does today
✓  Solid Overview — 8 stats + a 14-day activity chart.
✓  Three clean read tables: Restaurants, Contributors, Observation log.
✓  Auth-gated (IAP), desktop-only, paginated, searchable restaurants.
What an operator still needs
✗  No moderation. Parent notes & reviews go live with no approve / reject / report queue.
✗  No data-quality tools. A bogus 90-min observation can't be flagged or excluded from a venue's median.
✗  No actions on rows. Can't edit a venue, merge duplicates, or disable a user from the table.
✗  Flat dashboard. Nothing tells the operator what needs attention today.
Proposed navigation — two items change
Overview ＋ Moderation new Restaurants + detail Contributors + detail Data & observations ← was "Observation log" Location seeding Security
